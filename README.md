# 🛡️ HIPR — Telegram MTProto прокси

Невидимый для ТСПУ MTProto прокси. Весь трафик выглядит как HTTPS к `telegram.org`. Снаружи сервер отдаёт настоящий сайт-обманку.

## Как это работает

```
Telegram-клиент ──► 443 (nginx) ──► SNI=telegram.org ──► mtg:2398 ──► Telegram DC
Браузер/ТСПУ    ──► 443 (nginx) ──► SNI=ваш домен    ──► nginx:8443 ──► сайт-обманка
```

ТСПУ видит обычный HTTPS с сертификатом `telegram.org`. Active probe получает в ответ реальный HTML сайта.

---

## Требования

- Ubuntu 22.04 / 24.04 (Debian 11+)
- Чистый VPS с белым IP
- Домен с A-записью на IP сервера
- Порт 443 свободен

---

## Установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ChernOvOne/HIPR/main/install.sh)
```

Скрипт спросит:
1. **Домен** — например `proxy.example.com`
2. **Email** — для Let's Encrypt
3. **Тема сайта** — blog / freelancer / coming-soon
4. **Telegram-бот** — для уведомлений (необязательно)

Время установки: ~2 минуты.

---

## Подключение в Telegram

После установки скрипт выведет ссылку вида:
```
tg://proxy?server=proxy.example.com&port=443&secret=ee...
```

Откройте ссылку на телефоне — Telegram спросит «Включить прокси?».

Или через меню: **hide → [2] Ссылка + QR-код**

---

## Управление

```bash
hide          # интерактивное меню
hide status   # быстрый статус всех сервисов
hide link     # показать ссылку для Telegram
hide logs     # live-лог mtg
hide restart  # перезапустить всё
hide qr       # QR-код для подключения
```

### Меню `hide`:

| # | Раздел | Что делает |
|---|--------|-----------|
| 1 | 📊 Статус | Сервисы, IP, CPU, RAM, срок сертификата |
| 2 | 🔗 Ссылка | tg:// ссылка, QR-код, отправка в бот |
| 3 | 🎨 Темы | Смена сайта-обманки |
| 4 | ⚙️ Сервисы | Перезапуск, смена DC |
| 5 | 📋 Логи | Просмотр и очистка логов |
| 6 | 🔧 Настройки | Секрет, сертификат, бот |
| 7 | 🔄 Обновление | Обновить mtg / hide / пакеты |
| 8 | 🗑️ Удаление | Удалить или переустановить |

## Структура на сервере

```
/opt/hipr/
├── bin/
│   ├── mtg              # MTProto прокси
│   └── watchdog.sh      # авто-смена DC
├── config/
│   ├── mtg.toml         # конфиг mtg
│   └── active_theme     # текущая тема
├── themes/
│   ├── blog/            # тема: блог разработчика
│   ├── freelancer/      # тема: фрилансер
│   ├── coming-soon/     # тема: скоро открытие
│   └── custom/          # ваши темы
├── logs/
│   ├── mtg.log
│   └── watchdog.log
└── config.env           # основной конфиг

/var/www/hipr/           # активный сайт-обманка
/usr/local/bin/hide      # CLI команда
```

## Темы сайта-обманки

Сменить тему: **hide → [3] Темы → [1] Выбрать из установленных**

| Тема | Описание |
|------|----------|
| `blog` | Технический блог (Linux, Go, DevOps) |
| `freelancer` | Портфолио разработчика |
| `coming-soon` | «Скоро открытие» с таймером |

Добавить свою: положите папку с `index.html` в `/opt/hipr/themes/custom/` или используйте **hide → [3] → [3] Добавить свою**.

## Переустановка и удаление

```bash
# Переустановить
hide uninstall  # → [1] Переустановить

# Полностью удалить
hide uninstall  # → [2] Полностью удалить
```

При удалении предлагается бэкап конфига в `/root/hipr_backup_*`.

---

## Watchdog (авто-смена DC)

Запускается каждые 60 секунд через systemd timer. Если текущий Telegram DC недоступен — автоматически переключается на рабочий и перезапускает mtg. При настроенном боте отправляет уведомление.

## Часто задаваемые вопросы

**Сайт открывается, но прокси не работает?**
```bash
hide status         # проверить статус сервисов
journalctl -u hipr-mtg -n 30 --no-pager  # лог запуска
```

**Сменить Telegram DC вручную:**
```bash
hide  # → [4] Сервисы → [6] Сменить DC
```

**Обновить сертификат:**
```bash
hide  # → [6] Настройки → [2] Обновить TLS сертификат
```

**Переустановить после сбоя:**
```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ChernOvOne/HIPR/main/install.sh)
# Скрипт обнаружит существующую установку и предложит очистить
```

## Компоненты

- **[mtg v2](https://github.com/9seconds/mtg)** — MTProto прокси с FakeTLS
- **nginx** + `libnginx-mod-stream` — SNI-роутинг на порту 443
- **Let's Encrypt** — TLS сертификат для сайта-обманки
- **fail2ban** — защита SSH
- **ufw** — файрвол (открыты 22, 80, 443)
